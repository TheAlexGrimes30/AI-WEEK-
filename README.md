# Сравнение LLM на практической задаче

## 1. Цель задания
Познакомиться с современными языковыми моделями, научиться работать с API и сравнить возможности трех различных LLM на практической задаче, связанной с темами AI Learning Week.

---

## 2. Задача 3: Генератор технических описаний для AI-проектов
**Выполнил:** Гопиенко Александр  
**Команда:** Экспонента  
**Время:** 2-3 часа  
**Сложность:** ⭐⭐ Низкая-Средняя

### Что нужно сделать
1. Создать 8–10 кратких описаний идей AI-проектов (2–3 предложения каждое).
2. Попросить LLM сгенерировать для каждого:
   - Развернутое техническое описание
   - Список необходимых технологий/библиотек
   - Основные этапы реализации (3–5 пунктов)
   - Оценку сложности (легко/средне/сложно)
3. Сравнить качество генерации у трех моделей

**Пример входа:**  
*"Система автоматического обнаружения лесных пожаров с помощью дронов и компьютерного зрения"*

**Критерии сравнения:**
- Техническая грамотность
- Полнота описания
- Практичность рекомендаций

## 7. Выводы

**Лучшая модель:** YaGPT  
**Почему:** Она демонстрирует высокую точность терминологии, полноту информации и практическую применимость,
практически без упрощений.  
**Что удивило:** YaGPT создаёт структурированные и детализированные описания проектов, часто предлагая полезные
этапы реализации и рекомендации по технологиям, что значительно облегчает практическое внедрение идей.
Gemini предлагает креативные и разнообразные идеи, но иногда упрощает детали реализации
и недооценивает сложность некоторых проектов, поэтому она не всегда обеспечивает полное понимание практических шагов.
Cerebras даёт понятные и логичные описания проектов с пошаговыми планами, однако она иногда упрощает детали
и этапы некоторых проектов, из-за чего их практическая применимость может быть ограничена по сравнению с YaGPT.

## 8. Запуск проекта
В папках CerebrasLLM, GeminiLLM и YandexLLM находся py-файлы с кодом и README описание моделей с примером их результатов.
Для запуска необходимо создать .env файл в директории проекта, где необходимо указать API-KEY для каждой LLM, 
а для YaGPT необходимо указать URI и URL. Необходимо создать и активировать виртуальное окружение
и установить зависимости.

python -m venv venv

venv\Scripts\activate

pip install -r requirements.txt
